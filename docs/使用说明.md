# 热点与趋势 - 使用说明

## 📋 功能概述

本系统是一个完整的桌面应用程序，用于自动采集、管理和分析热点与趋势的文章。

### 核心功能
1. ✅ **自动定时采集** - 每 30 分钟自动采集最新文章
2. ✅ **增量存储** - 自动去重，只保存新文章
3. ✅ **搜索筛选** - 支持关键词和时间范围筛选
4. ✅ **导出分享** - 支持 JSON、Markdown 格式导出
5. ✅ **数据统计** - 作者统计、日期统计等分析功能

---

## 🚀 快速开始

### 1. 首次使用 - 登录并保存 Cookie

```bash
python login_save_cookies.py
```

- 系统会弹出浏览器窗口
- 手动登录热点与趋势账号
- 登录成功后，Cookie 会自动保存到 `cookies.json`

### 2. 启动管理程序

```bash
python article_manager_gui.py
```

- 程序启动后会自动执行第一次采集
- 之后每 30 分钟自动采集一次
- GUI 界面会显示所有采集的文章

## 📖 功能详解

### 1️⃣ 定时自动采集

- **启动时采集**：程序启动时立即执行一次采集
- **定时采集**：每 30 分钟自动采集一次
- **增量存储**：自动检测已存在的文章，只保存新文章
- **后台运行**：采集在后台线程执行，不影响界面操作

**采集日志**：
```
[2026-03-20 09:40:37.972173] 执行定时采集...
✓ 已读取 50 篇现有文章
✓ 新增 10 篇文章，总计 60 篇
```

### 2️⃣ 手动采集

点击界面上的 **"🔄 采集最新 10 篇"** 按钮，可以立即采集最新文章。

### 3️⃣ 搜索和筛选

**搜索框**：
- 输入关键词实时搜索
- 支持标题、作者、内容全文搜索

**筛选类型**：
- 全部：显示所有文章
- 今日：显示今天发布的文章
- 本周：显示最近 7 天的文章
- 本月：显示最近 30 天的文章

### 4️⃣ 查看文章

- **单击**：在右侧查看文章详情
- **双击**：在浏览器中打开原文
- **底部按钮**：可以打开原文、导出 Markdown、复制全文

### 5️⃣ 导出和分享

**导出全部文章**：
- 点击 **"📤 导出"** 按钮
- 选择 JSON 或 Markdown 格式
- 保存到指定位置

**导出单篇文章**：
- 选中一篇文章
- 点击 **"导出为 Markdown"** 按钮
- 保存为独立的 Markdown 文件

**复制内容**：
- 选中文章
- 点击 **"复制全文"** 按钮
- 内容自动复制到剪贴板

### 6️⃣ 数据统计

点击 **"📊 数据统计"** 按钮，查看：
- **总体统计**：文章总数、作者数量
- **作者统计**：Top 10 作者排名
- **日期统计**：每日发文数量

## 📁 数据文件说明

| 文件名 | 说明 |
|--------|------|
| `articles_database.json` | GUI 程序使用的文章数据库 |
| `broadcast_full.json` | 采集的完整原始数据 |
| `broadcast_full.md` | 采集文章的 Markdown 格式 |
| `broadcast_summary.txt` | 采集文章的摘要 |
| `cookies.json` | 登录凭证（敏感文件，不要分享） |
| `manager_config.json` | 程序配置（定时任务设置等） |

## ⚙️ 配置说明

### 定时任务设置

1. 点击菜单 **"工具" → "定时任务设置"**
2. 勾选 **"启用定时采集"**
3. 设置采集时间（如 `09:00`）
4. 点击 **"保存设置"**

> 注意：当前版本固定为每 30 分钟采集一次，定时设置功能预留扩展。

## 🔧 技术细节

### 增量存储机制

1. **读取现有数据**：启动时加载 `articles_database.json`
2. **URL 去重**：基于文章 URL 判断是否重复
3. **合并数据**：新文章在前，旧文章在后
4. **保存更新**：同时更新多个数据文件

### 采集流程

```
启动程序
    ↓
加载现有文章
    ↓
访问热点与趋势首页
    ↓
查找广播消息列表
    ↓
遍历最新 10 篇文章
    ↓
对每篇文章：
    - 提取标题、作者、时间
    - 访问详情页提取完整内容
    - 检查 URL 是否已存在
    ↓
保存新增文章
    ↓
更新 GUI 显示
```

## ❓ 常见问题

### Q1: 采集失败怎么办？
**A**: 检查以下几点：
1. 确认 `cookies.json` 文件存在
2. 如果 Cookie 过期，重新运行 `python login_save_cookies.py`
3. 检查网络连接是否正常

### Q2: 如何停止定时采集？
**A**: 关闭程序即可停止定时采集。

### Q3: 数据文件可以删除吗？
**A**: 
- `articles_database.json`：可以删除，但会丢失所有文章数据
- `cookies.json`：不要删除，否则需要重新登录
- `broadcast_full.*`：可以删除，下次采集会重新生成

### Q4: 可以修改采集频率吗？
**A**: 当前版本固定为 30 分钟。如需修改，编辑 `article_manager_gui.py` 第 549 行：
```python
schedule.every(30).minutes.do(self.schedule_collect)  # 修改 30 为其他分钟数
```
