# 韭研公社文章管理器 - 目录结构规划

## 一、推荐目录结构

```
JC/
├── src/                          # 源代码目录
│   ├── article_manager_gui.py    # GUI 主程序
│   ├── collect_broadcast_stable.py # 采集脚本
│   └── login_save_cookies.py     # 登录工具
│
├── data/                         # 数据文件目录
│   ├── database/                 # 数据库文件
│   │   └── articles_database.json # 文章数据库
│   ├── cookies/                  # Cookie 文件
│   │   └── cookies.json          # 登录凭证
│   └── export/                   # 导出文件
│       ├── json/                 # JSON 导出
│       └── markdown/             # Markdown 导出
│
├── logs/                         # 日志目录
│   ├── collect_2026-03-20.log    # 采集日志
│   ├── app_2026-03-20.log        # 应用日志
│   └── error_2026-03-20.log      # 错误日志
│
├── config/                       # 配置文件目录
│   ├── manager_config.json       # 主配置
│   └── scheduler_config.json     # 定时任务配置
│
├── tools/                        # 工具脚本
│   └── clean_project.py          # 清理工具
│
├── docs/                         # 文档目录
│   ├── README.md                 # 项目说明
│   ├── 使用说明.md                # 详细使用指南
│   └── user_rules.md             # 项目规则
│
└── temp/                         # 临时文件目录
    └── page_debug.html           # 调试文件
```

## 二、各目录功能说明

### 1. src/ - 源代码
- 存放所有 Python 源代码文件
- 便于版本管理和代码维护
- 可以添加 `__init__.py` 变成 Python 包

### 2. data/ - 数据文件
**data/database/**
- 存储文章数据库文件
- 定期备份历史数据

**data/cookies/**
- 存储登录凭证
- 敏感文件，需要加入 `.gitignore`

**data/export/**
- 用户导出的文章
- 按格式分类存储

### 3. logs/ - 日志文件
- 自动记录采集日志
- 应用运行日志
- 错误日志（便于调试）
- 按日期分割，便于查找

### 4. config/ - 配置文件
- 存储所有配置项
- 便于用户修改设置
- 支持多套配置

### 5. tools/ - 工具脚本
- 清理工具
- 数据迁移工具
- 批量处理工具

### 6. docs/ - 文档
- 项目文档
- 使用说明
- 开发文档

### 7. temp/ - 临时文件
- 调试输出
- 临时数据
- 可安全删除

## 五、迁移步骤

### 1. 创建目录结构
```bash
mkdir -p src data/database data/cookies data/export/json data/export/markdown
mkdir -p logs config tools docs temp
```

### 2. 移动文件
```bash
# 源代码
mv article_manager_gui.py src/
mv collect_broadcast_stable.py src/
mv login_save_cookies.py src/

# 数据文件
mv articles_database.json data/database/
mv cookies.json data/cookies/

# 工具
mv clean_project.py tools/

# 文档
mv README.md docs/
mv 使用说明.md docs/
mv user_rules.md docs/
```

### 3. 创建配置文件
- 在 config/ 目录创建配置文件
- 使用默认配置初始化

### 4. 更新代码路径引用
- GUI 程序读取配置
- 采集脚本使用新路径
- 日志模块指向 logs/

## 六、优势

1. **结构清晰**：功能分区明确，易于理解
2. **便于维护**：代码、数据、日志分离
3. **易于备份**：data/ 目录整体备份
4. **日志追踪**：完整的操作记录
5. **配置灵活**：支持多套配置
6. **安全隔离**：敏感文件单独存放
7. **易于扩展**：新增功能不影响现有结构

## 七、后续优化建议

1. **添加日志轮转**：自动清理旧日志
2. **数据备份**：定期备份 database/
3. **配置验证**：启动时检查配置有效性
4. **错误恢复**：日志记录错误并支持恢复
5. **性能监控**：记录采集耗时等指标
